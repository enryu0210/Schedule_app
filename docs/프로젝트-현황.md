## 5. 핵심 결정 & 함정 (중요)

- **카카오 KOE205 해결**: Supabase(GoTrue)의 카카오 기본 scope가
  `account_email profile_image profile_nickname`로 **고정**되어 클라이언트에서 제거 불가.
  → 해결: 카카오 **"개인 개발자로 등록"**(사업자등록증 없이 본인인증)으로 `account_email` 잠금 해제,
  동의항목 3개 모두 **"선택 동의"**, Supabase Kakao provider의 **"Allow users without an email" ON**.
  (앱은 이메일 불필요 — 카카오 고유 ID로 식별)
- **Vercel-저장소 연결**: 한때 Vercel이 **다른 저장소**를 배포해 코드가 반영 안 됨.
  → Settings → Git 에서 `enryu0210/Schedule_app` 연결 확인 필수.
- **환경변수**: `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY` (Vite는 **빌드 시점**에 주입 →
  변경 후 **캐시 없이 재배포** 필요). Vercel + 로컬 `web/.env` 양쪽 필요.
- **남은 테스트 데이터**: enryu0210 계정엔 이전 마이그레이션으로 생긴 '방학' 프리셋이 클라우드에 남아 있음
  (신규 사용자만 빈 상태). 편집 모드 → '프리셋 삭제'로 제거 가능.
