## 1. 프로젝트 개요

**일정공방** — 요일별 시간 블록으로 주간 계획을 짜고, 상황별(방학·학기중·휴가) **프리셋**으로 저장해
여러 기기에서 한눈에 보는 앱. "지금 할 일"을 자동 하이라이트.
바탕화면에 띄워두는 **데스크탑 위젯**으로도 본다.

- **GitHub**: https://github.com/enryu0210/Schedule_app.git (branch: `main`)
- **웹 배포**: Vercel (Root Directory = `web`) → `schedule-repo-eight.vercel.app` (main 푸시 시 자동 배포)
- **위젯 배포**: GitHub Releases (`widget-v*` 태그) + 앱 내 자동 업데이트
- **디자인 톤**: 테라코타/크림 따뜻한 톤

- **자동 업데이트** (`tauri-plugin-updater`) — 서명 검증 후 설치·재시작

```
├─ docs/
│  ├─ 2단계-Supabase-카카오-설정.md
│  ├─ 위젯-설계.md              # ★ 위젯 설계·릴리스 절차·함정
│  ├─ AI-시간표-비용-산정.md
│  └─ 프로젝트-현황.md          # (이 문서)
├─ desktop/                    # ★ 데스크탑 위젯 (Tauri)
│  └─ src-tauri/src/
│     ├─ lib.rs                # 부팅 + 플러그인(위치기억/자동실행/업데이터) 등록
│     ├─ tray.rs               # 트레이 아이콘 + 메뉴

└─ web/
   └─ src/
      ├─ App.tsx               # 라우터: 위젯모드 → 로딩 → LoginScreen → Planner
      ├─ types.ts              # Preset / DayPlan / ScheduleBlock
      ├─ styles/tokens.css     # ★ 색상 토큰(테마 수정 지점)
      ├─ hooks/
      │  ├─ useAuth.tsx        # 세션/카카오 로그인·로그아웃
      │  ├─ usePresetStore.tsx # ★ 클라우드 전용 프리셋 저장소 (읽기+쓰기)
      │  └─ useWidgetPresets.ts# 위젯 전용 읽기 전용 로더 (캐시 → Realtime + 폴링)
      ├─ lib/
      │  ├─ cloudStorage.ts    # fetch(예외 던짐)/load(삼킴)/saveCloudPresets
      │  ├─ widgetCache.ts     # ★ 위젯 오프라인 캐시 (localStorage, 사용자별)
      │  ├─ widgetMode.ts      # 위젯 모드 판별 + 우클릭 차단
      │  └─ widgetWindow.ts    # 로그인 동안 창 확대 → 복원
      └─ components/
         ├─ Planner.tsx        # ★ 메인 계획표
         ├─ WidgetView.tsx     # 위젯 컨테이너(로그인/로딩)
         ├─ WidgetBody.tsx     # 위젯 화면(순수 컴포넌트)
         ├─ DownloadWidget.tsx # 위젯 다운로드 버튼
         └─ (PresetSidebar, ScheduleCard, WeekGrid, BlockEditor, AuthBar …)
```

**데이터 흐름**
- 웹: `useAuth`(세션) → `usePresetStore`(클라우드 읽기/쓰기) → `Planner`
- 위젯: `useAuth` → `useWidgetPresets`(**읽기 전용**) → `WidgetBody`
  웹에서 저장하면 Supabase **Realtime** 이 위젯에 즉시 알려준다 (폴링 60초는 안전망)

## 5. 핵심 결정 & 함정 (중요)

- **카카오 KOE205**: 카카오 "개인 개발자로 등록" + 동의항목 전부 선택 동의 +
  Supabase Kakao provider 의 "Allow users without an email" ON 으로 해결.
- **OAuth 리다이렉트는 쿼리를 지키게 한다.** `redirectTo` 를 `window.location.origin` 으로 주면
  위젯 창에서 로그인했을 때 `?widget=1` 이 날아가 **위젯에 전체 앱 화면이 뜬다**(실제로 겪음).
  안전망으로 위젯 모드를 sessionStorage 에도 기억시킨다.
- **위젯은 절대 클라우드에 쓰지 않는다.** 읽기 전용 훅(`useWidgetPresets`)을 따로 둔 이유 —
  저장 로직을 달고 다니면 오래된 상태로 웹의 최신 수정을 덮어쓸 수 있다.
- **"데이터 없음"과 "못 읽음"은 다르다.** `loadCloudPresets` 는 오류를 삼키고 `null` 을 돌려주기 때문에,
  위젯이 이걸 그대로 쓰면 네트워크가 끊긴 순간 `presets = []` 로 덮어써져 빈 화면이 됐다.
  그래서 예외를 던지는 `fetchCloudPresets` 를 따로 두고, 위젯은 실패하면 화면을 건드리지 않는다.
- **Vercel-저장소 연결**: 한때 Vercel 이 다른 저장소를 배포해 코드가 반영 안 됨.
  Settings → Git 에서 `enryu0210/Schedule_app` 연결 확인.
- **환경변수**: `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY` (Vite 는 **빌드 시점** 주입 →
  변경 후 캐시 없이 재배포 필요). Vercel + 로컬 `web/.env` 양쪽 필요.
- **위젯 관련 함정**은 `docs/위젯-설계.md` 에 모아뒀다 (MSVC 필수 / Git Bash 링커 충돌 /
  latest.json 의 BOM / PowerShell 의 `$env:VAR=""` 는 변수 삭제).

## 6. 개발 환경 (새 PC 에서 시작할 때)

```powershell
# 웹
cd web; npm install; npm run dev      # http://localhost:5173 (web/.env 에 Supabase 키 필요)
cd web; npm run build                 # 타입체크 + 프로덕션 빌드

# 위젯 (Rust + MSVC 필요 — rustup 은 MSVC 를 자동으로 깔아주지 않는다)
winget install --id Rustlang.Rustup -e
winget install --id Microsoft.VisualStudio.2022.BuildTools -e --override "--wait --quiet --add Microsoft.VisualStudio.Workload.VCTools --includeRecommended"
cd desktop; npm install; npm run dev   # 위젯 창 띄우기 (반드시 PowerShell 에서 — Git Bash 는 링커가 깨짐)
```

**위젯 서명 키**: `~/.tauri/schedule-widget.key` (+ `.pass`).
저장소 밖에 있고 **잃어버리면 다시는 업데이트를 밀어넣을 수 없다. 반드시 백업.**
