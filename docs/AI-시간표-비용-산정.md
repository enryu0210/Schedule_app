> 작성일: 2026-07-13 / 기준 가격: Anthropic Claude API 공식 단가
> 목적: "AI에게 자연어로 할일을 알려주면 시간표를 짜주는" 기능의 운영 비용 추정

## 1. 요청 1회의 크기 가정

| 항목 | 토큰 | 근거 |
|------|------|------|
| 입력 — 시스템 프롬프트 + JSON 스키마 + 기존 프리셋 맥락 | ~1,500 | 블록 구조 설명 + 출력 형식 지정 |
| 입력 — 사용자 자연어 요청 | ~500 | "평일 오전엔 논문, 오후엔 근로..." |
| **입력 합계** | **~2,000** | |
| 출력 — 주간 시간표 JSON (7일 × 약 5블록 = 35블록) | **~1,500** | 블록당 30~40토큰(한글 라벨 포함) |

> 한글은 영어보다 토큰 소모가 크므로 넉넉하게 잡은 값이다.

## 4. 결론 & 설계 지침

- **모델은 Haiku 4.5로 시작한다.** 시간표 생성은 "자연어 → 정해진 JSON 구조" 변환 작업이지
  깊은 추론이 필요한 일이 아니다. Structured Outputs(JSON 스키마 강제)를 쓰면 형식이 깨지지 않는다.
  품질이 아쉬우면 Sonnet으로 올려도 여전히 월 수만 원대다.

- **프롬프트 캐싱은 이 기능에 도움이 안 된다.** 캐싱은 *입력* 비용을 깎는 기능인데
  이 기능은 비용의 약 80%가 *출력* 토큰에서 나온다. 게다가 Haiku 4.5의 최소 캐시 프리픽스는
  4,096토큰이라 우리 시스템 프롬프트(~1,500)는 캐싱이 아예 걸리지 않는다.
  → 비용을 줄이려면 **출력을 줄여라**. (예: 색상은 서버에서 규칙으로 부여하고 AI는 시간·할일만 생성)

- **⚠️ 진짜 리스크는 비용이 아니라 API 키 노출이다.**
  Vite는 환경변수를 **빌드 시점에 번들에 박아 넣으므로**, 프론트에서 Claude API를 직접 호출하면
  개발자도구로 키를 꺼내갈 수 있다 → 비용 무한대.
  → **반드시 Vercel 서버리스 함수를 한 겹 두고**, 키는 서버에만, 사용자당 호출 횟수 제한을 건다.

- **온디바이스 SLM(miniLM)은 폐기.** miniLM은 문장 임베딩 모델이라 시간표를 *생성*할 능력이 없다.
  브라우저에서 돌릴 수 있는 생성 모델은 수백 MB 다운로드가 필요해 웹앱 UX와 상극이다.
  월 1만 회에 14만 원이면 서버 API가 압도적으로 합리적이다.
