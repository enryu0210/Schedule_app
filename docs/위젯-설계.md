# 위젯 설계 (데스크탑 우선)

> 작성일: 2026-07-13
> 목표: "수시로 확인할 수 있는 위젯" — 지금 무슨 일정을 하고 있는지 항상 보이게 한다.

---

## 1. 요구사항 (원 스펙)

- **데스크탑(윈도우 우선)**: 바탕화면 위에 띄워놓는 형태. **Win+D를 눌러도 내려가지 않아야 함.**
- **안드로이드**: 홈 화면 위젯 + 네비게이션/알림 바에서 현재 진행 중인 일정 확인.

---

## 2. 기술 선택 — 왜 Tauri인가

| | Tauri (선택) | Electron |
|---|---|---|
| 상주 메모리 | 30~60MB (윈도우 내장 WebView2 사용) | 150~200MB (Chromium 통째로 탑재) |
| 배포 파일 | ~10MB | ~150MB |
| Win32 API 호출 | Rust에서 정식 지원(`windows` 크레이트) | `ffi-napi`/`koffi` 같은 FFI 우회 필요 |
| 설치 부담 | Rust + VS 빌드 도구 필요(일회성) | Node만 있으면 됨 |

**위젯은 하루 종일 켜져 있는 프로그램**이다. 상주 메모리가 결정적이라 Tauri를 택했다.

---

## 3. Win+D를 이겨내는 방법

일반적인 "항상 위(always-on-top)" 창은 Win+D(바탕화면 보기)에 **같이 내려간다.** 두 가지 해법이 있다.

1. **WorkerW 자식으로 붙이기** — 창을 바탕화면 레이어의 자식으로 만들어 "바탕화면의 일부"가 되게 한다.
   Win+D의 영향을 아예 안 받지만 **바탕화면 아이콘 뒤에 깔린다.**
2. **`WM_WINDOWPOSCHANGING` 가로채기** — Win+D는 창을 화면 밖(-32000, -32000)으로 밀어내는 방식으로
   동작한다. 이 메시지를 가로채 그 이동을 무시하면 창이 그대로 남는다. **아이콘 위에 뜬다.** ← **채택**

> 기존 라이브러리(electget, tauri-plugin-wallpaper)는 모두 취미 수준이고,
> tauri-plugin-wallpaper 저자조차 "취미가 아니면 Win32 로직을 직접 구현하라"고 권고한다.
> → **Win32 호출은 우리가 직접 작성한다.**

---

## 4. 구조 — 데이터는 절대 두 벌로 관리하지 않는다

순진하게 접근하면 데스크탑 앱에 **카카오 로그인 + Supabase 클라이언트 + 동기화 로직을 새로 붙이게 된다.**
같은 코드를 두 벌 관리하는 셈이고, 버그가 두 배로 생긴다.

**대신 Tauri 창이 이미 배포된 웹앱을 그대로 로드한다.**

```
┌─ Tauri (Rust) — "창의 성질" ──────────────┐
│  · 테두리 없는 투명 창                      │
│  · 항상 위 + Win+D 무시 (Win32)            │
│  · 드래그로 이동, 트레이 아이콘             │
│                                            │
│   └─ WebView2 ─────────────────────────┐  │
│      https://<배포주소>/?widget=1       │  │
│      → 로그인 세션·데이터 동기화가        │  │
│         웹앱 그대로 공짜로 따라옴          │  │
│      → 우리가 만들 건 WidgetView 화면뿐   │  │
│      ────────────────────────────────┘  │
└────────────────────────────────────────────┘
```

- **웹 쪽 할 일**: `?widget=1` 일 때 렌더링되는 컴팩트한 `WidgetView` (← 1단계, Rust 없이 가능)
- **Rust 쪽 할 일**: 창의 성질 + Win32 (← 2단계, Rust 설치 후)

**트레이드오프**: 오프라인이면 빈 화면이 된다. 추후 로컬 캐시로 보완한다.

## 5. WidgetView 화면 사양

- 지금 진행 중인 일정을 **크게** (없으면 "지금은 일정이 없어요")
- 다음 일정 2개를 작게
- 현재 시각 / 오늘 요일
- 데이터는 **읽기 전용** — 위젯은 절대 클라우드에 쓰지 않는다.
  (웹에서 수정한 내용을 반영하려고 60초마다 다시 읽기만 한다)
- 비로그인 상태면 로그인 안내만 노출

## 7. 폴더 구조

```
desktop/
├─ package.json          # @tauri-apps/cli 만 (프론트엔드 빌드 없음)
├─ public/index.html     # 자리표시용 (실제 화면은 배포된 웹앱을 로드)
└─ src-tauri/
   ├─ tauri.conf.json    # 창 속성 + 위젯 URL
   ├─ capabilities/      # 원격 URL(배포 웹앱)에 창 이동 권한 부여
   └─ src/
      ├─ lib.rs          # 앱 부팅 + 창 핸들 얻어 Win32 적용
      └─ win_desktop.rs  # Win+D 무시 (서브클래싱)
```

## 8. 사전 준비 (개발 환경)

```
winget install --id Rustlang.Rustup -e
winget install --id Microsoft.VisualStudio.2022.BuildTools -e --override "--wait --quiet --add Microsoft.VisualStudio.Workload.VCTools --includeRecommended"
```

**함정 2가지 (실제로 겪음):**
1. **rustup은 VS C++ 빌드 도구를 자동으로 깔아주지 않는다.** Tauri는 MSVC 링커가 필수라
   위 두 번째 명령을 반드시 따로 실행해야 한다.
2. **Git Bash / MSYS 에서 `cargo build` 를 하면 링크가 깨진다.** `C:\MinGW\msys\1.0\bin\link.exe`
   (GNU coreutils의 link)가 MSVC `link.exe` 를 가려버리기 때문이다.
   → **PowerShell 에서 빌드할 것.**

## 9. 빌드 / 실행

```powershell
cd desktop
npm run dev     # 개발 모드로 위젯 창 띄우기
npm run build   # 설치 파일(NSIS) 생성 → src-tauri/target/release/bundle/
```
